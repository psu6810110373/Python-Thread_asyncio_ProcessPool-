# โปรแกรมทดสอบการทำงานพร้อมกัน (Concurrency) ใน Python

โปรเจกต์นี้เป็นการทดสอบเขียนโปรแกรมเพื่อดูการทำงานของเครื่องมือ 3 ตัวในภาษา Python ได้แก่ `asyncio`, `threading`, และ `multiprocessing` โดยเลือกใช้ตามความเหมาะสมของประเภทงาน 

1. วิธีการ asyncio จากไฟล์ 01Asyncio.py
    ไฟล์นี้คือการใช้ asyncio เพื่อให้โปรแกรมไม่หยุดการทำงานเพื่อรอ response จาก sever แต่จะข้ามไปรันคำสั่งถัดไปพลางๆ และเมื่อ server ส่งข้อมูลกลับมาเสร็จ โปรแกรมถึงจะกลับไปรับและจัดการข้อมูลนั้น ทำให้โปรแกรมสามารถทำงานหลายๆ คำสั่งให้เสร็จพร้อมกันได้เร็วขึ้นในโปรแกรมตัวเดียว

2. วิธีการ threading จากไฟล์ 02Threads.py
    ไฟล์นี้คือการใข้ Thread เพื่อแบ่งการทำงานและช่วยให้โปรแกรมสามารถสลับการคำนวณไปมาได้ โดยจากโค้ด มีการสั่งให้สร้างสายการทำงานขึ้นมา 3 สาย ในจังหวะที่สายการทำงานที่ 1 รันมาถึงคำสั่ง with open(...) และกำลังรอให้ฮาร์ดดิสก์เขียนข้อมูลบรรทัดแรกให้เสร็จ ระบบปฏิบัติการจะสลับให้สายการทำงานที่ 2 และ 3 เตรียมข้อความบรรทัดถัดไปรอไว้เลย โดยการทำงานที่แบ่งไปนั้น จะยังแชร์พื้นที่ความจำเดียวกัน และมีกฎของโปรแกรมจำกัดไว้ว่า สามารถประมวลผลคำสั่งได้ทีละสายเท่านั้น ไม่สามารถทำงานขนานกันแบบ 100% ได้

3. วิธีการ multiprocessing จากไฟล์ 03ProcessPool.py
    ไฟล์นี้คือการใข้ multiprocessing จากโค้ดมีการใช้ลูป for ที่คำนวณ hashlib.sha256 ถึง 5 แสนรอบ เป็นงาน CPU Bound แท้ๆ ที่ไม่มีการรอ I/O เลยหากใช้แบบ thread จะช้ามาก เพราะไม่สามารถทำแบบขนานได้ เราจึงใช้ multiprocessing เพิ่อสร้าง Process ระดับ OS ขึ้นมาใหม่ 4 ตัว แต่ละ Process ได้รับหน่วยความจำและ Python Interpreter ของตัวเอง  ทำให้รันโค้ดชุดนี้บน CPU หลายๆ คอร์แบบขนานกันได้
